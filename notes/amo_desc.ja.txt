コンテキストメニュー（右クリック）から、お好みのエディタを使用して、「ソース表示」「選択部分のDOMツリー表示」、または「テキスト編集」ができるようになります。

<em>ソースを表示</em>
<ul>
  <li>(X)HTML、JavaScript、CSS、SVG、XMLなどのソースをエディタで開きます。</li>
  <li>ローカルファイルを参照している場合、直接、そのローカルファイルを開きます。</li>
</ul>
<em>選択部分のDOMツリーを表示</em>
<ul>
  <li>選択部分のDOMツリーを生成してXMLで表示します。ソースのDOMツリーとは必ずしも一致しません。</li>
  <li>選択部分がDOMで解析不能な場合は、選択部分のみを含んだファイルを生成して表示します。</li>
</ul>
<em>テキストを編集</em>
<ul>
  <li>input要素・textarea要素や編集可能な要素上で有効になります。</li>
  <li>エディタで編集・保存した内容は、ブラウザに戻って元の要素にフォーカスを移すと同期されます。</li>
</ul>
* * *
<em>アドオンマネージャでの設定</em>
<em>エディタの選択</em>
<ul>
  <li>使用するエディタを登録します。</li>
  <li>Mac OS Xでは、 <code>/Applications/[yourEditor].app/Contents/MacOS/</code> 内にあるファイルを選択してください。</li>
</ul>
<em>コマンドラインオプション</em>
<ul>
  <li>エディタの起動オプションを利用する場合に設定します。</li>
  <li>バックスラッシュ（\）文字にはバックスラッシュを加えてエスケープさせておく必要があります。
      例: <code>C:<em>\</em>\Windows</code></li>
  <li>空白が含まれる場合は二重引用符で囲ってください。
      例: <code><em>"</em>Some Arg<em>"</em></code></li>
</ul>
<em>DISPLAY環境変数</em>
<ul>
  <li>Linux専用。"Cannot open display"などのエラーが出る場合に設定してください。別途、xhostの設定も必要かもしれません。</li>
</ul>
<em>コンテキストメニュー上のアクセスキー</em>
<ul>
  <li>アクセスキーは任意の値に変更可能です。ただし、一文字である必要があります。</li>
</ul>
<em>プライベートブラウジング中も有効化</em>
<ul>
  <li>チェックを入れると有効、チェックを外すと無効です。</li>
</ul>
<em>終了時に一時ファイルを強制的に削除</em>
<ul>
  <li>プライベートブラウジングの終了時やブラウザの終了時に一時ファイルを強制的に削除するかどうか選択できます。チェックを入れておくことを推奨しますが、エディタによっては強制削除を有効にすると不具合が起こる可能性があります。</li>
</ul>
<em>アドオンの無効化時に全設定を初期化</em>
<ul>
  <li>チェックを入れると、アドオンの無効化ボタンを押した際に、すべての保存データが消去され、設定が初期化されます。</li>
</ul>
* * *
<em>ツールバーボタンでの設定</em>
<ul>
  <li>アイコンの色を変更したり、エディタの表示名を変更できます。また、ここからアドオンマネージャを開くことができます。</li>
</ul>
